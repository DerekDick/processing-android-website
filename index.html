The Processing for Android project is an effort within the Processing community to bring up-to-date support for Android development into the Processing environment. The main component of this project is the Android Mode, a programming mode for the Processing Development Environment (PDE) that allows you to create Android apps using the Processing language. With this mode, you can run Processing sketches in Android devices with little or no changes in the code, while also being able to access the Android SDK for advanced functionality (sensors, geolocation, etc.)

The mode takes care of repetitive tasks such as installing the SDK files, creating project folders, and setting default assets. All you need to do is to write your sketch as you would normally do when working in the Java mode, and then simply run it on your Android device or the emulator. You can also export your sketch as a signed package to upload to the Google Play Store.

The current version of the mode (3.0) allows you to create regular Android apps. The upcoming 4.0 version, currently in beta stage, adds support for live wallpapers, watch faces, and Google Cardboard. Stay tuned for updates from the Contribution Manager in the PDE, or the Processing twitter channel!


Credits

Work on the Android mode started in 2010 with Ben Fry and Jonathan Feinberg creating the basic pieces, and Andres Colubri developing the GLES renderer. During Google Summer of Code 2014, Imil Ziyaztdinov implemented signed package export, device selection, and automatic SDK download. Shortly afterwards, Umair Khan updated the mode to work in Processing 3, refactored the core library to use Fragments, and wrote a video playback and capture library, as part of Google Summer of Code 2015. In 2016, with support from Google Creative Lab, Andres Colubri extended the mode to include live wallpapers, watch faces, and Cardboard apps. For a full list of contributors, check this page.
